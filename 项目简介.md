# Web3 智能资金行为分析（项目简介）

## 项目定位

本项目聚焦 AI 代币赛道，目标是从链上地址行为中识别高置信 Smart Money 候选地址，并将地址资金流转化为可解释的风险信号，辅助判断市场情绪和资金拥挤程度。

## 研究问题

在高波动市场中，价格本身往往滞后。  
本项目关注的核心问题是：

**如何基于链上资金行为，稳定筛选高质量地址，并形成可复盘、可解释的群体画像。**

## 研究范围与口径

- 赛道：AI 代币
- 标的：`FET (Ethereum)` + `RNDR (Solana, 趋势对比)`
- 时间窗口：`45 天滚动`
- 分析粒度：日级地址资金流

Smart Money 候选地址的基础口径：

- `active_days >= 10`
- `net_flow_usd > 0`
- `net_position_token > 0`
- `min_net_flow_usd >= 1000`
- 日级净流入达到 `p80` 分位阈值（动态门槛）

## 项目产出内容

- Dune Dashboard（KPI + 趋势图 + 分层图）
- 可复用 SQL 查询集合（候选池、成本模型、PnL 分层）
- 项目说明文档（口径、结论、局限）

## 核心能力与亮点

- 双链统一分析口径：将 Ethereum 与 Solana 数据统一到同一分析框架。
- 候选筛选规则可复现：使用可量化规则组合，而非主观挑选地址。
- 结果解释链条完整：从候选池规模到成本收益，再到群体分层，能够连续解释资金状态。
- 展示形式友好：结果已沉淀为可交互 Dashboard，便于演示与复盘。

## 业务价值

- 帮助识别“资金扩张/收缩”阶段，补足价格指标滞后问题。
- 提供群体胜率与收益分层视角，支持风险判断与仓位讨论。
- 作为 Web3 数据分析项目，具备可展示、可复述、可追问的完整证据链。

## 局限性说明

- 地址标签体系并不完美，仍可能存在误分类风险。
- 未实现 PnL 为模型估算口径，不等同税务级精确核算。
- 不同链的流动性结构存在差异，跨链对比以趋势解释为主。
